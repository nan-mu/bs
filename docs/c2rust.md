# C2Rust 转换规则

当用户要求c2rust时，记住只读被转换文件，读取其余文件会扰乱结果。你应该是没用户提出的每一个rs文件创建一个Todo，然后让干净的接口以此处理Todo

## 转换前

在转换前，需要使用wc -l命令检查函数行数，若超过100行，警告用户并暂停。

## 转换规则

1. Rust 函数应该永远返回anyhow的Result，对于c的void函数，返回Result<()>。但某些函数一定会返回某个值或是不可能报错，那么不用Result。
2. C语言有时候返回一个常量形如`return -EXXXX`，应该转换为`tail!(-EXXXX)`。
3. 只应该转换函数，不涉及其他语言结构。
4. 对于较大函数（你自己感觉什么是较大函数）暂停，提示用户需要拆分。
5. 尽量使用可变引用替代指针。尽量避免使用指针。
6. 所有的变量采取就近原则。
7. 数字运行统一采用防溢出函数。

## 转换后

1. 使用如下命令，将已有函数信息插入数据库。

```shell
/Users/nan/bs/tools/c2rust_existing.fish <转换文件路径>
```

2. 使用如下命令，查询缺少类型。

```shell
/Users/nan/bs/tools/c2rust_missing.fish <转换文件路径>
```

输出格式如下，你需要将这些缺少内容放置文档注释(`//!`)。
函数：xxx,xxx,xxx
类型：xxx,xxx,xxx
值：xxx,xxx,xxx
库：xxx,xxx,xxx

如果文档注释已有对缺失内容的描述，你应该删除已有描述。
